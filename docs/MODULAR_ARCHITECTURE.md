# Modular Architecture Design

## 🏗️ Overview

The Demo Builder uses a **modular plugin architecture** where:
- **Framework** provides stable base classes and interfaces
- **LLM generates modules** that implement these interfaces
- **Modules are saved** as Python files in project subdirectories
- **Loader dynamically executes** modules at runtime

## 🎯 Benefits

### 1. Maintainability
- Framework code remains stable
- Each demo is isolated in its own module
- Updates don't break existing demos

### 2. Collaboration
- Modules can be shared between team members
- Version controlled in Git
- Can be improved over time

### 3. Customization
- Generated modules can be manually refined
- Add customer-specific logic post-generation
- Combine automated and manual improvements

### 4. Reusability
- Successful demos become templates
- Modules can be cloned and adapted
- Build a library of industry patterns

## 🔌 Base Classes (Framework)

### DataGeneratorModule
```python
class DataGeneratorModule(ABC):
    @abstractmethod
    def generate_datasets(self) -> Dict[str, pd.DataFrame]:
        """Generate demo datasets"""

    @abstractmethod
    def get_relationships(self) -> List[tuple]:
        """Define foreign key relationships"""
```

### QueryGeneratorModule
```python
class QueryGeneratorModule(ABC):
    @abstractmethod
    def generate_queries(self) -> List[Dict[str, Any]]:
        """Generate ES|QL queries"""

    @abstractmethod
    def get_query_progression(self) -> List[str]:
        """Define presentation order"""
```

### DemoGuideModule
```python
class DemoGuideModule(ABC):
    @abstractmethod
    def generate_guide(self) -> str:
        """Generate demo guide"""

    @abstractmethod
    def get_talk_track(self) -> Dict[str, str]:
        """Query-specific talk track"""
```

## 🤖 Module Generation Process

1. **User provides context** (company, pain points, use cases)

2. **LLM generates Python code** implementing base classes:
   ```python
   # Generated by LLM for Salesforce
   class SalesforceDataGenerator(DataGeneratorModule):
       def generate_datasets(self):
           # Custom logic for Salesforce data
           accounts = pd.DataFrame({
               'account_id': [...],
               'health_score': [...],  # Specific to their pain point
               'churn_risk': [...]     # Addresses their use case
           })
           return {'accounts': accounts}
   ```

3. **Module saved to demos directory**:
   ```
   demos/salesforce_customer_success_20241021/
   ├── data_generator.py
   ├── query_generator.py
   └── demo_guide.py
   ```

4. **Loader executes module** dynamically

## 🔄 Workflow

### Creating a New Demo
```python
from src.framework import create_demo

config = {
    'company_name': 'Acme Corp',
    'department': 'Sales',
    'pain_points': ['Slow reporting'],
    'use_cases': ['Sales Analytics']
}

results = create_demo(config)
# Creates: demos/acme_corp_sales_20241021/
```

### Running an Existing Demo
```python
from src.framework import run_demo

results = run_demo('acme_corp_sales_20241021')
datasets = results['datasets']
queries = results['queries']
guide = results['demo_guide']
```

### Customizing a Module
1. Navigate to `demos/module_name/`
2. Edit any Python file
3. Re-run the demo to see changes

## 🎨 Customization Points

### Data Patterns
Edit `data_generator.py` to:
- Add more columns
- Adjust data distributions
- Include specific anomalies
- Change data volume

### Query Refinement
Edit `query_generator.py` to:
- Optimize query performance
- Add new queries
- Adjust complexity progression
- Fix any ES|QL issues

### Demo Flow
Edit `demo_guide.py` to:
- Update talk track
- Add industry insights
- Customize objection handling
- Refine the narrative

## 🚀 Advanced Features

### Module Inheritance
Create specialized modules that extend base modules:
```python
from demos.salesforce_base.data_generator import SalesforceDataGenerator

class SalesforceHealthcareDataGenerator(SalesforceDataGenerator):
    def generate_datasets(self):
        datasets = super().generate_datasets()
        # Add healthcare-specific data
        datasets['patient_accounts'] = self.generate_patient_data()
        return datasets
```

### Module Composition
Combine components from different modules:
```python
from demos.module1.data_generator import Module1DataGenerator
from demos.module2.query_generator import Module2QueryGenerator
```

### Module Templates
Save successful modules as templates:
```bash
cp -r demos/great_demo/ templates/industry_template/
```

## 📊 Benefits Over Static Code

| Static Approach | Modular Approach |
|----------------|------------------|
| One-size-fits-all | Customer-specific |
| Hard to customize | Easy to tailor |
| Breaks on updates | Isolated from changes |
| Limited reuse | Highly reusable |
| No version control | Git-tracked modules |

## 🔍 Example: Salesforce Module

The LLM generates this for Salesforce Customer Success:

### data_generator.py
```python
class SalesforceDataGenerator(DataGeneratorModule):
    def generate_datasets(self):
        # Specific to Salesforce's 5000 accounts, $10B ARR
        accounts = pd.DataFrame({
            'account_id': [f'ACC-{i:06d}' for i in range(5000)],
            'arr': np.random.lognormal(14, 1.5, 5000),  # $10B total
            'health_score': self._generate_health_scores(),
            'last_qbr': self._generate_qbr_dates(),
            'churn_risk': self._calculate_churn_risk()
        })

        usage_metrics = pd.DataFrame({
            'account_id': np.repeat(accounts['account_id'], 30),
            'date': pd.date_range(end=datetime.now(), periods=150000),
            'api_calls': np.random.poisson(1000, 150000),
            'active_users': np.random.poisson(50, 150000)
        })

        return {
            'accounts': accounts,
            'usage_metrics': usage_metrics
        }
```

```python
                'expected_insight': 'Weekly health score trends'
            }
        ]
```
