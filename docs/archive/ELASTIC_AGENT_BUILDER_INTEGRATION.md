## 🛠️ Tool Metadata Generation

### Automatic Generation During Module Creation

Each query now includes tool metadata generated by the LLM:

```python
"tool_metadata": {
    "tool_id": "company_dept_purpose",  # Max 50 chars, lowercase
    "description": "Analyzes X. Identifies Y for Z.",  # Action-focused
    "tags": ["tag1", "tag2", "esql"]  # 3-5 relevant tags
}
```

### Tool ID Pattern

```
<company>_<department>_<purpose>
```

Examples:
- `jpmc_infra_cpu_performance`
- `tmobile_network_capacity_analysis`
- `salesforce_customer_churn_prediction`

### Generation Prompt Strategy

The LLM receives:
1. Customer context (company, department, industry)
2. Pain points and use cases
3. Query purpose and description
4. Strict formatting rules for tool IDs

## 🎨 UI Implementation

### Browse Demos → Agents & Tools Tab

#### Tools Subtab

**Sections**:

1. **Deployed Tools**
   - Lists tools matching module prefix
   - View JSON, delete functionality
   - Shows filtering criteria

2. **Tool Candidates**
   - Validated queries from current module only
   - Query type indicators (SCRIPTED, PARAMETERIZED, RAG)
   - Tool metadata forms with pre-filled values
   - API call preview before deployment

**Filtering Logic**:
```python
module_prefix = get_module_prefix(module_name)  # e.g., "jpmc_infra"
deployed_tools = [t for t in all_tools if t['id'].startswith(module_prefix)]
```

#### Agents Subtab

**Sections**:

1. **Deployed Agents**
   - Shows agents matching module
   - Currently assigned tools list
   - Tool assignment interface
   - Multi-select for module + platform tools

2. **Agent Configuration**
   - Pre-populated from `agent_metadata.json`
   - Editable fields for all properties
   - Save locally or deploy to Agent Builder
   - Handles create/update based on existence

**Tool Assignment**:
- Checkboxes for available tools
- Pre-selected currently assigned tools
- Separate sections: Module tools | Platform tools
- Single update replaces entire tool list (API requirement)

## 🔍 Module Prefix Extraction

The system uses intelligent prefix extraction for filtering:

```python
def get_module_prefix(module_name: str) -> str:
    # 1. Try agent_metadata.json (most reliable)
    if agent_id.endswith('_agent'):
        return agent_id[:-6]  # Remove '_agent' suffix

    # 2. Try tool_metadata.json (fallback)
    # Extract from any tool_id pattern

    # 3. Parse module name (last resort)
    parts = module_name.split('_')[:2]
    return f"{parts[0]}_{parts[1]}"
```
