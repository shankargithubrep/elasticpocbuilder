# Demo Guide: Itaú Unibanco - Customer Experience

## Overview
**Industry:** Banking/Financial Services
**Focus Areas:** Inability to connect frontend navigation with backend requests through distributed tracing, Difficulty in defining 'satisfactory performance' - each application and business has unique performance definitions, challenging to establish universal benchmarks based on Core Web Vitals

## Demo Flow
1. **Introduction** - Establish context around their key challenges
2. **Data Exploration** - Show 7 targeted queries addressing pain points
3. **Business Impact** - Highlight speed, accuracy, and insights gained

## Key Queries
1. Frontend-to-Backend Performance Correlation via Distributed Tracing
2. Navigation Timing Phase Breakdown for Performance Diagnosis
3. Core Web Vitals Monitoring by Application Type & Business Context
4. Resource Loading Optimization with CDN Performance Analysis
5. Long Task & JavaScript Error Correlation Analysis

## Value Proposition
- **Speed:** Insights in seconds vs hours/days
- **Scalability:** Handles growing data volumes
- **Self-Service:** Empowers teams with intuitive query language

## Closing Points
- ES|QL provides SQL-like simplicity with Elasticsearch power
- Queries run directly on indexed data (millisecond response times)
- Most teams productive within days, not weeks
